## Smart Contract Compilation

Contract File -> Solidity Compiler -> ABI + Bytecode

Bytecode -> Ethereum Blockchain

**ABI**

ABI is a Abstract Binary Interface. Suppose there is a contract B which want to talk to our contract A then it will need an ABI of A.

Without ABI we can not  talk to A.

So Now we can see that whole contract does not run on the Ethereum Chain rather only the bytecode runs on the Ethereum Chain.

Bytecode is a JSON which contains all the information about the contract.

ABI will have all the functions which are available in the contract. 

**Note:** 

- Contract Bytecode is public in readable format.
- Contract does not have to be public.
- Bytecode is immutable
- ABI act as a brigde between the applications and the smart contract.
- ABI and Bytecode can not be generated without the smart contract source code.
